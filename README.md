## 📌 Overview

This project analyzes operations for a fictional hospital network,
**Northwind General Health**. Using a realistic relational dataset of
patients, encounters, diagnoses, providers, and billing, it answers the
kind of questions a healthcare data analyst is asked every day:

- Who are our patients, and how is the population changing?
- How long do patients stay, and which departments are busiest?
- What is our **30-day readmission rate** and where is it worst?
- How much do we bill vs. collect, and which payers reimburse best?

All data is **synthetic** and generated by script, it contains **no real
patient information (PHI)**.

## 🧩 Data model

Eight related tables, organized around the **encounter** (a hospital visit)
as the central fact table:

```
departments ──┐
              ├──< providers ──┐
patients ─────┼────────────────┼──< encounters ──┬──< diagnoses
              │                                   ├──< procedures
              │                                   ├──< prescriptions
              └───────────────────────────────────┴──── billing (1:1)
```

| Table           | Rows   | Description                                  |
|-----------------|-------:|----------------------------------------------|
| `patients`      |  2,000 | Demographics, location, insurance            |
| `providers`     |     60 | Doctors and their department/specialty       |
| `departments`   |     12 | Clinical departments                         |
| `encounters`    |  4,675 | Hospital visits (admit/discharge, type)      |
| `diagnoses`     | 11,665 | ICD-10 coded diagnoses per encounter         |
| `procedures`    |  7,127 | Procedures performed                         |
| `prescriptions` |  9,330 | Medications prescribed                        |
| `billing`       |  4,675 | Charges, payments, and payment status        |

See [`sql/01_schema.sql`](sql/01_schema.sql) for full column definitions,
primary keys, foreign keys, and check constraints.

## 🚀 How to run it

### Option A — PostgreSQL (the canonical version)

```bash
# 1. Create a database
createdb healthcare

# 2. Build the schema, load data, add indexes
psql -d healthcare -f sql/01_schema.sql
psql -d healthcare -f sql/02_load_data.sql      # run from the repo root
psql -d healthcare -f sql/03_indexes.sql

# 3. Run any analysis file
psql -d healthcare -f sql/analysis/03_readmissions.sql
```

### Option B — Zero setup (Python + SQLite)

No database server? Reproduce every headline number instantly:

```bash
python scripts/run_analysis_sqlite.py
```

### Regenerate the data (optional)

```bash
python scripts/generate_data.py     # deterministic; re-creates data/*.csv
```

## 📈 Selected findings

A few highlights from [`findings/insights.md`](findings/insights.md):

- **30-day readmission rate is 10.2% overall**, peaking in **Oncology
  (13.6%)** and **Orthopedics (12.3%)** — the top targets for intervention.
- **$90.1M billed / $83.3M collected = 92.5% collection rate.** Self-Pay
  patients collect at just **75.8%** vs. 92%+ for insured payers.
- **Cardiology and Oncology have the longest stays** (≈6-day median, 11-day
  p90), driving both cost and bed utilization.
